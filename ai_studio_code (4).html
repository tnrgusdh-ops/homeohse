LLM의 정의 및 파이프라인 개요

거대 언어 모델(LLM)은 텍스트 상자 뒤에 숨겨진 거대한 예측 기계로, 사용자가 입력한 단어들에 반응하여 다음 단어를 생성하는 마법 같은 도구입니다. 안드레 파시(Andrej Karpathy)는 이 도구가 특정 분야에서는 매우 뛰어나지만 다른 분야에서는 한계가 명확하다는 점을 강조하며, 전체적인 구축 파이프라인에 대한 정신적 모델을 제공합니다.


  • 텍스트 기반 예측 모델로서의 정체성: 사용자 입력에 따라 다음에 올 단어들을 확률적으로 계산하고 생성하는 확률적 텍스트 생성 시스템으로서의 역할.
  • 전체 구축 파이프라인의 단계별 구조: 사전 훈련, 지도 미세 조정, 강화 학습으로 이어지는 일련의 정교한 모델 제작 공정에 대한 포괄적 이해.
  • 모델의 장점과 한계에 대한 균형 잡힌 인식: 특정 작업에서의 놀라운 마법 같은 성능과 더불어 주의해야 할 날카로운 단점들에 대한 비판적 시각 견지.
  • 인간과 AI 대화의 본질적 의미 파악: 단순히 텍스트를 주고받는 것을 넘어 모델이 학습한 데이터의 통계적 패턴이 어떻게 발현되는지에 대한 통찰.

1단계: 사전 훈련 - 데이터 수집 및 필터링

LLM 구축의 첫 번째이자 가장 거대한 단계는 인터넷 전체를 다운로드하여 처리하는 사전 훈련 단계입니다. 오픈AI, 구글 등은 'FineWeb'과 같은 거대한 데이터셋을 사용하여 인터넷의 방대한 정보를 모델 내부에 내재화합니다. 44테라바이트에 달하는 텍스트 데이터가 필터링을 거쳐 고밀도 정보로 압축됩니다.

원시 데이터인 Common Crawl 데이터는 매우 거칠기 때문에 고품질의 데이터셋을 만들기 위해 URL 필터링, 텍스트 추출, 언어 필터링, 개인정보 제거 등의 복잡한 전처리 과정을 거칩니다. 이는 모델이 불필요한 노이즈를 배우지 않도록 하는 핵심적인 품질 관리 공정입니다.

토큰화의 원리

텍스트를 신경망에 입력하기 전, 텍스트를 1차원적인 기호 시퀀스로 변환하는 토큰화 과정이 필요합니다. 컴퓨터는 비트를 처리하지만, 효율성을 위해 'BPE(Byte Pair Encoding)' 알고리즘을 사용하여 자주 등장하는 바이트 쌍을 하나의 토큰으로 묶습니다.

GPT-4는 약 10만 개의 고유 기호를 사용하여 텍스트를 효율적으로 압축합니다. 토큰화 과정에서 대문자와 공백 유무에 따라 전혀 다른 ID가 부여되며, FineWeb과 같은 데이터셋은 약 15조 개의 토큰 시퀀스로 변환되어 모델 학습의 원료로 사용됩니다.

2단계: 지도 미세 조정(SFT)

베이스 모델을 유용한 '비서(Assistant)'로 바꾸기 위해 SFT 단계를 거칩니다. 인터넷 문서를 흉내 내는 대신 인간이 작성한 이상적인 대화 데이터를 학습시켜 질문에 답하는 방식을 가르칩니다. 이 단계는 사전 훈련에 비해 훨씬 적은 데이터와 시간(예: 몇 개월 대신 몇 시간)이 소요되지만, 모델의 페르소나를 결정하는 핵심 단계입니다.

기업들은 전문 레이블러를 고용하여 도움됨, 진실함, 무해함(HHH) 원칙에 따라 이상적인 대화를 작성하게 합니다. 이를 통해 모델은 단순한 텍스트 생성을 넘어 전문가의 연구 과정과 사고 방식을 통계적으로 흉내 내는 비서 모델로 진화합니다.

3단계: 강화 학습(RL)과 추론의 등장

기존의 SFT는 인간의 답변을 단순히 모방하는 데 그쳐, 모델이 스스로 최적의 사고 경로를 발견하기 어려웠습니다. 강화 학습(RL)은 모델이 다양한 시도를 하고 정답 여부에 따라 보상을 받음으로써 스스로 학습하게 하는 단계입니다. 이는 마치 학생이 교과서의 연습 문제를 스스로 풀며 정답지에 맞춰보며 실력을 키우는 과정과 유사합니다.

최근 DeepSeek R1과 OpenAI의 o1 모델은 이러한 RL 과정을 통해 '사고(Thinking)' 능력을 극대화한 추론 모델입니다. 이들은 정답을 내놓기 전 내부적으로 긴 '사고의 사슬(CoT)'을 전개하며, 스스로 가설을 세우고 검증하며 오류를 수정합니다. 이를 통해 수학, 코딩 등 복잡한 논리 문제에서 비약적인 성능 향상을 이뤄냈습니다.

미래 기술 전망

미래의 LLM은 텍스트를 넘어 오디오, 이미지, 비디오를 직접 이해하고 생성하는 '멀티모달(Multimodal)' 모델로 진화할 것입니다. 또한, 단순한 답변을 넘어 사용자를 대신해 작업을 수행하는 '에이전트(Agents)'로서 컴퓨터를 직접 조작하는 기능을 갖추게 될 것입니다.

하지만 LLM은 여전히 오류가 있을 수 있는 도구입니다. 사용자는 모델을 맹신하기보다 오류를 인식하고 책임감 있게 사용해야 합니다. 인간이 감독자로서 AI 에이전트를 지휘하는 새로운 협업 형태가 미래의 업무 환경이 될 것입니다.
